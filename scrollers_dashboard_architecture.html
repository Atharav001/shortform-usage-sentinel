Scroller's Dashboard Architecture

Android App | Database-First Architecture | Real-time Intervention System

Background Engine

  • • ScrollerAccessibilityService runs 24/7
  • • Monitors TYPE_WINDOW_STATE_CHANGED events
  • • Batches scrolls (every 3) to save battery
  • • Syncs with UsageStatsManager every 30s
  • • Platform trackers: Instagram (RecyclerView), YouTube (ViewPager)

Database-First Architecture

  • • Room Database is the single source of truth
  • • 4 tables: ScrollRecords, ScrollEvents, UserSettings, Tasks
  • • Kotlin Flow enables real-time UI updates
  • • Service and UI both read/write to same DB
  • • No memory state—everything persisted

Intervention System

  • • WindowManager overlay blocks addictive apps
  • • Triggered when daily limit or alert frequency hit
  • • Displays real-life goals from Tasks table
  • • GLOBAL_ACTION_HOME closes distracting app
  • • Context redirection: addiction → productivity
