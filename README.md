📱 Short-Form Sentinel: Structural Usage TrackerA precision-engineered Android Accessibility Service designed to monitor consumption habits on Instagram Reels and YouTube Shorts. Unlike traditional trackers that rely on intrusive screen recording or simple app-foreground timers, this project uses UI Structural Analysis and Physics-Based Debouncing to count actual content swipes with 99% accuracy.⚠️ The Problem: The "Signal vs. Noise" ChallengeTracking short-form video is notoriously difficult because standard Android event listeners cannot distinguish between:Intentional Swipes: Moving from one video to the next.System Noise: Micro-stutters or "wiggling" a video without switching.UI Interference: Scrolling through comments, profile feeds, or search results that use the same underlying scroll components as the video player.Early iterations using simple scroll listeners resulted in massive overcounting, often triggering 5-10 events for a single video.🛠️ The Solution: Architectural EvolutionThe app's logic evolved through rigorous testing to move from "Event-Based" tracking to "Behavior-Based" validation.1. Structural Detection (Component Filtering)Instead of monitoring raw pixels or generic scroll events, the service targets specific UI engines:Targeting: RecyclerView and ViewPager components.Why: These are the specific containers used for vertical video paging. By ignoring generic View or ScrollView events, we immediately filter out noise from Settings menus and DM lists.2. The 70% "Golden Ratio" RuleTo separate the full-screen video player from half-screen elements (like comment sections or expandable descriptions), we implemented a geometric threshold:The Logic: A scroll is only validated if the moving container covers at least 70% of the screen height.The Calibration: * 50% Threshold: Too sensitive (triggered by comments).90% Threshold: Too restrictive (failed due to System Status/Navigation bars).70% Result: The "Sweet Spot" that stays resilient across different aspect ratios while ignoring non-video UI.3. 900ms "Physics Debouncing"To solve the "Micro-Scroll" problem—where a single physical flick triggers multiple system events—we implemented a One-at-a-Time cooldown:The Threshold: 900ms (Instagram) and 800ms (YouTube).The Reasoning: It takes roughly 300-500ms for a human to complete a swipe and for the UI to "snap." By blinding the app for ~900ms after a valid trigger, we eliminate jitter while remaining fast enough to catch "speed-scrollers."⚙️ Technical ImplementationFeatureImplementation DetailLanguage/ToolingKotlin, Android Accessibility Suite, Android Studio AIDetection EngineAccessibilityEvent.TYPE_VIEW_SCROLLED with Class Name filteringGeometry Validationnode.getBoundsInScreen(rect) comparison against DisplayMetricsEfficiencyBuffered Writes: Counts are cached and synced every 3 scrolls/seconds to minimize CPU wake-locks.Development ProcessThis application was developed using Android Studio AI as a pair-programmer. The logic wasn't written in one pass; it was an iterative journey of refinement:Iterative Logic: Used Local History to track and revert logic phases (from 2-minute cooldowns to physics-based debouncing).Real-world Debugging: Captured UI node trees from various devices to calibrate the 70% height rule against diverse system overlays.🔒 Privacy & EthicsPrivacy-First by Design: This app is built on the principle of data minimization.No Screen Recording: The app never captures screenshots or video. It "sees" the structure of the UI, not the content of the images.Local Processing: All structural metadata (height checks, class names) is processed on-device.Transparency: The Accessibility Service is used strictly for geometric validation of scroll containers, ensuring the user's visual data remains private.🚀 Future RoadmapDigital Wellbeing Integration: Setting "Hard Caps" that lock the specific containers after a certain count.Cross-Platform Parity: Expanding the 70% rule to TikTok and other emerging short-form platforms.
